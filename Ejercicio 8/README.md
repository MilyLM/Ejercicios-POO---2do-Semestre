# Ejercicio 8: Sistema de Torneo (POO)

Este proyecto implementa un sistema de gestión para un torneo utilizando **Programación Orientada a Objetos** en Python. El objetivo es administrar diferentes tipos de perfiles (Competidores y Observadores) y sus interacciones con el sistema de puntuación.

## Estructura del Código

El ejercicio se divide en los siguientes módulos:
* `Competidor.py`: Define la clase para los participantes activos.
* `Observador.py`: Define la clase para los usuarios que visualizan el torneo.
* `Jugador.py`: Clase base o auxiliar para la lógica de usuarios.
* `main.py`: Punto de entrada que simula el flujo del torneo con ejemplos de uso.

## 📝 Ejemplo de Ejecución

El programa realiza las siguientes acciones:
1. Crea un competidor llamado "Carlos Méndez".
2. Suma y resta puntos según su desempeño en el torneo.
3. Crea un observador llamado "Ana Torres" que registra partidas vistas.
4. Muestra un resumen detallado de ambos perfiles en consola.
